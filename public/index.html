API reference

All /api/v1 routes require Authorization: Bearer <api-key>.

POST /api/v1/payments Create payment (idempotent)
GET /api/v1/payments List all payments
GET /api/v1/payments/:id Get payment by ID
PATCH /api/v1/payments/:id/status Update payment status
GET /health Liveness probe
GET /ready Readiness (DB + Redis)
Example — create a payment
POST /api/v1/payments
Authorization: Bearer dev-api-key
Idempotency-Key: order_abc_001
Content-Type: application/json

{
  "amount": 1000,
  "customerId": "cust_123"
}

→ 201 Created
